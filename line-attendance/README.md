# line-attendance — LINE 請假 / 交接 / 代理人（完整版增益模組）

把「請假登記、假單 OCR、工作交接、代理人、SOP 問答」整合進一個公司 LINE 官方帳號。**只有要請假功能才需要這個資料夾**；純知識庫只裝 `core/`。

## 包含
- `n8n-flows/leave-bot.json` — 主機器人（已去敏，含佔位符）
- `n8n-flows/sop-create.json` — SOP 投稿流程（LIFF 表單 → webhook → 待審）
- `liff-form/sop-liff-form.html` — SOP 投稿用 LIFF 表單

## 核心設計原則
- **LLM 只做「聽懂訊息→JSON」與「講人話」**；所有判斷（算日期、找紀錄、去重、路由）都是 Code 節點裡的 deterministic JavaScript。
- **群組**負責：請假登記、查詢、修改/取消、假單圖片。
- **私訊**負責：身分綁定、填交接單、查交接、查代理人、SOP 問答（`/sop`）。
- **存取閘門**：私訊非成員只能走「綁定 / 新成員」，其餘不回應；群組訊息不受影響。

## 機器人指令（使用者打 `/help` 會看到）
- 第一次：私訊「綁定 暱稱」（現有同仁）／「新成員」自助建檔（新人）
- 請假（群組）：`姓名 日期 假別`，例「小明 6/25 特休」；或直接傳假單截圖
- 查詢：「今天誰請假」「這週請假」
- 修改/取消（群組，帶日期）：「小明 6/25 改成事假」「取消 小明 6/25 的假」
- 交接（私訊）：填交接單回傳；查交接「小明 6/25 的交接」；查代理人「小明 的代理人是誰」
- SOP（私訊，每句帶 `/sop`）：「/sop 報帳要準備哪些文件」；建立「/sop建立」

## 進階機制（已內建於 leave-bot.json）
- **假單真偽辨識**：上傳假單圖片時，先讓視覺模型判斷「外觀」（表單截圖／紙本表單／純文字／照片）；純文字冒充、風景照疊字、或出現「假的／測試／偽裝」字眼一律擋下並回嗆，不寫入。
- **時數以假單為準**：OCR 直接讀假單「扣假時數」欄的數字（最優先），看不到才用上下班時間自算，避免午休扣抵造成誤差。
- **判重複看「時數相加」、不看假別**：同人＋日期重疊＋兩筆時數相加 > 8 才算同一個假（重複→擋/合併）；相加 ≤ 8 視為不同時段（上午＋下午），兩筆都留。**刻意不比對假別名稱**，因為「特休 / 遞延特休 / 補休」可能是同一個假但寫法不同，比對假別會抓不到。登記偵測、背景去重、假單比對三處邏輯一致。
- **交接同步代理人**：請假人填回交接單後，自動把交接內容私訊轉發給「代理人」（代理人需已綁定）；並提示代理人有問題先用 `/sop` 查、真的查不到再問本人。
- **發訊息前提**：所有主動私訊（交接單模板、轉發代理人）都需對方「已綁定 userId 且加 bot 好友」，否則靜默跳過（並提醒填寫者）。
- **回覆功能改假（userId 後備）**：LINE「回覆」原請假訊息來改假時，webhook 只給被回覆訊息 ID、拿不到原文 → 若解析不到姓名，改用「發話者 userId 對到本人」找出那筆假；未綁定者提示先綁定。
- **時數讀「天數×8＋時數」**：假單以「扣假天數＋扣假時數」或「共X天Y時」表示時，總時數＝天數×8＋時數（1天=8小時）；避免把「1天」誤判成「1時」。兩欄都讀不到才退回用上下班時間推算。
- **SOP 行為式延續（sticky）**：私訊打一次 `/sop` 進入後，後續訊息直接當 SOP 追問（不用每句帶 /sop）；做別的明確功能（請假/綁定/交接…）自動接管並退出；純打招呼走閒聊；10 分鐘沒互動自動結束。用 n8n 跨執行記憶（`$getWorkflowStaticData`）按 userId 記狀態，try/catch 防呆。**註：跨執行記憶只在 workflow Active 時持久化，編輯器手動 Execute 不會留。**
- **SOP 追問帶上下文（followup）**：sticky 追問時 body 帶 `followup:true`，後端據此帶該 sessionId 最近數輪＋脈絡化檢索（解「說多一點/繼續」這類指代不足的追問）。上下文嚴格按 sessionId（=各人 LINE userId）隔離，不同人不會串；查無資料仍誠實回。
- **假單讀圖（filesystem 相容）**：用 n8n 官方 `getBinaryDataBuffer()` 取圖片 buffer，記憶體／filesystem 兩種 `binaryDataMode` 都能正確拿到圖；避免 filesystem 模式下 `binary.data` 為空、把空圖送進視覺模型導致解析失敗。讀不到 JSON 時安靜跳過、不報紅錯。
- **日期解析**：支援「今天／明天／後天／昨天／前天」「下週X（具體某天）」「兩天／三天等中文數字＋N天（→日期範圍）」；「下週／本月」這類沒指定哪天的模糊詞會反問日期、不亂用今天。
- **天數顯示**：整天倍數顯示「N 天」、不足一天顯示「N 小時」、混合顯示「N 天 M 小時」。
- **假單補登近 14 天**：假單日期在過去但 14 天內、且無對應既有假 → 仍可補登（補昨天的假單）；超過 14 天的舊單才略過。純文字「補假單／補單」（沒寫假別）不會被當成新請假亂登記。
- **改假防 AI 腦補人名**：要改誰的假，名字必須真的出現在訊息原文，否則一律當「改發話者本人」；避免 LLM 從請假紀錄 JSON 撈出沒提到的人名。
- **相對時數改假**：「＋N小時／多N／再N」＝原時數加 N；「－N／少N」＝減 N；「改成 X 小時」＝直接設定。

## 接線重點（詳見根目錄 INSTALL.md Part 5–6）
1. n8n 匯入兩個 json → 設 Google Sheets 憑證 → 換掉所有 `<佔位符>`。
2. Google 試算表分頁：`leaves` / `employees`(A本名 B暱稱 C LINE_UserID) / `handover`(A–E：姓名/請假期間/內容/更新時間/代理人)。
3. LINE channel token、群組 ID（白名單）、SOP 後端位址（預設 localhost:3345）都要填。
4. LIFF 表單填 `<LIFF_ID>`、`<N8N_HOST>`，托管到 HTTPS。

> ⚠️ 佔位符一覽：`<LINE_CHANNEL_ACCESS_TOKEN>`、`<GOOGLE_SHEET_ID>`、`<LINE_GROUP_ID>`、`<N8N_GOOGLE_SHEETS_CREDENTIAL_ID>`、`<LOCAL_CHAT_API_KEY>`、`<LIFF_ID>`、`<N8N_HOST>`、`<SOP_GENERATOR_FORM_URL>`。全部換成你自己的值。
